# 0008 - The Ten Workstreams (scaffold expansion)

**Date:** 2026-07-05 · **Status:** in flight · **Owner:** the producer (execution), vision (scope)

The product passed its structural audits (identity: shopping engine for AI
agents; live hunt moment; Free/Pro/Max). This cycle expands the scaffolding so
every surface is coherent with the Item-first model and the operator ICP, via
ten parallel bounded workstreams (Sonnet agents, strict file ownership):

1. **Onboarding** - first-run speaks operator language, saves first finds as Items, carries hunt intent.
2. **Item detail** - /wishlist/item/[id] view/edit; grid links to it.
3. **Radar -> Items** - scan results land as wish-list Items (title/url/price), not seller records.
4. **Agent item tools** - built-in chat agent gets save_item/list_items/mark_item_purchased/lists.
5. **Connect guides** - per-client setup on /connect (Claude Code, Cursor, Codex, OpenClaw, Hermes, JSON).
6. **Tests** - item ops, demo-hunt helpers exported + tested, API validation.
7. **SEO** - per-page metadata, JSON-LD, sitemap completeness.
8. **A11y/perf** - marketing sections: aria, alt, focus, reduced-motion audit.
9. **Settings UX** - API-key create/copy flow with inline connect snippet.
10. **Dashboard empty states** - first-visit paths point to connect + Radar.
